How Hardware Data Reaches This App

This system is designed for an ESP32 + UV sensor station. The hardware sends UV readings to this web app API, then the backend validates, classifies risk, stores records, and updates all dashboards.

  1. Sense: UV sensor connected to ESP32 reads current UV index.
  2. Transmit: ESP32 sends JSON to POST /api/readings over Wi-Fi.
  3. Secure ingest: Backend checks payload format and API key.
  4. Classify: Server maps UV to risk: Low (0-2.99), Moderate (3-5.99), High (6+).
  5. Store: Reading is saved in SQLite; alerts are auto-created for Moderate/High levels.
  6. Visualize: Frontend pages poll APIs every few seconds to display live updates.

If physical hardware is unavailable, use demo mode from API: POST /api/demo/start.
