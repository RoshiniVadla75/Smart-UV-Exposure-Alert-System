How This System Works

This page explains the real hardware-to-web data flow for the Smart UV Exposure Alert System. The ESP32 reads UV sensor values and sends them securely to this Flask backend, which processes and serves data to all dashboard pages.

1) UV Sensing on ESP32

The hardware station (ESP32 + UV sensor) measures UV index at regular intervals. It also captures metadata like signal strength (`wifi_signal`) and timestamp.

2) Send Reading to Backend API

ESP32 sends an HTTP POST request to /api/readings with JSON data and X-API-Key header. 

{
  "device_id": "uv-station-01",
  "uv_index": 4.8,
  "wifi_signal": -62,
  "timestamp": "2026-04-23T10:30:00Z",
  "source": "hardware"
}

3) Validation + Security

Backend validates payload fields, checks timestamp format, verifies API key, and applies rate limiting/security headers for safer public usage.

4) Risk Classification + Storage

Server computes risk from UV index: Low (0-2.99), Moderate (3-5.99), High (6+). Then it stores readings in SQLite and creates alert records for Moderate/High events.

5) Live UI Updates

Frontend pages poll API endpoints to render current UV, risk, history, alerts, and device status. This keeps the dashboard near real-time for users.

6) Demo Mode (No Hardware)

If hardware is unavailable, the built-in demo engine can generate realistic sample readings via /api/demo/start and stop via /api/demo/stop.
